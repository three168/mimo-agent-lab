# MiMo Agent Lab

一个面向工程资料审查与本地自动化流程的轻量 Agent 实验仓库。

这个 Demo 的目标不是做一个普通聊天应用，而是验证大模型在真实长链任务中的可用性：

- 把复杂目标拆成可执行步骤
- 为每一步生成检查点和回退条件
- 对代码、日志、配置草案和风险项进行复核
- 将过程整理成可追溯报告

## Planned MiMo Usage

我计划把 Xiaomi MiMo-V2.5 接入到本地 Agent 工作流中，用于以下方向：

1. 工程项目资料审查  
   例如硬件项目中的电源、复位、接口、外围电容、材料温度特性等检查项。

2. 本地自动化流程辅助  
   例如整理项目资料、生成检查清单、分析运行日志、复核脚本草案和输出阶段报告。

3. 多 Agent 协同  
   - Planner Agent：拆解任务与生成执行计划
   - Executor Agent：生成脚本或处理流程草案
   - Reviewer Agent：复核风险点、边界条件和回退条件
   - Reporter Agent：整理日志、结论和待确认问题

## Demo

```bash
python src/agent_plan_runner.py
```

输出会模拟一个长链工程任务的 Agent 编排过程：任务拆解、风险复核、执行计划和报告生成。

## Why MiMo

这个项目重点测试 MiMo-V2.5 在以下场景中的表现：

- 长链推理
- 多 Agent 协同
- 工程文档理解
- 代码与脚本草案生成
- 日志和结果分析
- 自我复核与风险提示

## Status

当前仓库是一个最小 Demo，用于展示工作流结构。后续计划接入 MiMo API，并补充更多测试记录、运行截图和 Prompt 模板。
